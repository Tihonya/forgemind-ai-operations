# ForgeMind AI Operations — Source of Truth

## Призначення пакета

Цей каталог є нормативним джерелом для проєктування, реалізації, перевірки та демонстрації портфоліо-проєкту **ForgeMind AI Operations**.

Проєкт створюється як доказ здатності пройти повний шлях:

> бізнес-проблема → аналіз процесу → технічні вимоги → архітектура → MVP → AI-агент → інтеграція → тестування → governance → публічний результат.

Усі бізнес-дані, документи, назви виробів, постачальників, співробітників та інцидентів — **синтетичні**. Проєкт не копіює внутрішні системи або конфіденційні дані будь-якої реальної компанії.

## Що саме ми будуємо

Ми будуємо не універсальну корпоративну AI-платформу і не симулятор роботів.

Перша завершена версія — це один повний вертикальний сценарій:

# Supply Risk Intelligence

Система аналізує синтетичний виробничий план, BOM, складські залишки та відкриті поставки, знаходить ризики дефіциту комплектуючих, шукає дозволені альтернативи в інженерній базі знань, формує пояснену рекомендацію, створює чернетку дії, вимагає підтвердження людини та зберігає повний audit trace.

## Головна метрика завершення

Проєкт вважається **Portfolio Ready**, лише коли одночасно виконані всі умови:

1. Публічний HTTPS URL працює на VPS.
2. Новий користувач може пройти основний demo-сценарій без ручного втручання розробника.
3. Усі acceptance tests з `04_ACCEPTANCE_TESTS.md` мають статус `PASS`.
4. Golden Scenario повертає рівно очікувані три ризики на незмінному seed dataset.
5. Розрахунок ризиків виконується детермінованим кодом, а не вгадується LLM.
6. RAG-відповіді містять перевірювані посилання на синтетичні документи.
7. Змінювальна дія не виконується без human approval.
8. Audit Log відтворює повний ланцюжок виконання.
9. `docker compose up -d --build` розгортає систему з чистого середовища.
10. Backend, frontend та end-to-end тести проходять.
11. Репозиторій містить актуальну документацію, архітектурну схему та інструкцію запуску.
12. Є коротке демонстраційне відео та стабільний demo-акаунт.

**Будь-який невиконаний пункт означає, що проєкт ще не завершений.**

## Правило зміни scope

Нову функцію дозволено додавати до MVP лише після явного запису в `08_DECISION_LOG.md` із відповідями:

- яку проблему вона вирішує;
- який acceptance test її перевіряє;
- що буде вилучено або відкладено, щоб не роздути scope;
- хто затвердив зміну.

Інакше функція автоматично належить до Post-MVP.
