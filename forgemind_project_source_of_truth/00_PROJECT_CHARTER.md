# 00. Project Charter

## 1. Робоча назва

**ForgeMind AI Operations**

Підзаголовок:

> AI-assisted supply risk analysis for engineering and manufacturing operations.

Назва є нейтральною та вигаданою. Вона не повинна імітувати бренд реальної компанії.

## 2. Мотивація

У зростаючій інженерно-виробничій компанії критична інформація розподілена між:

- ERP;
- виробничими планами;
- BOM;
- складськими залишками;
- закупівлями;
- технічною документацією;
- журналами рішень;
- повідомленнями між командами.

Через це працівники витрачають час на ручне зіставлення даних, а ризик зриву виробничого плану може бути виявлений запізно.

## 3. Проблема

Менеджер виробництва або закупівель не має єдиного інструмента, який:

1. детерміновано визначає дефіцит комплектуючих;
2. враховує відкриті поставки та строки;
3. знаходить затверджені альтернативи у документації;
4. пояснює причини ризику;
5. формує наступну дію;
6. залишає критичне рішення за людиною;
7. зберігає аудит усіх кроків.

## 4. Ціннісна пропозиція

ForgeMind скорочує шлях від нового виробничого плану до поясненого переліку ризиків та контрольованого плану дій.

Система демонструє поєднання:

- process mapping;
- Python development;
- React/TypeScript;
- ERP integration;
- RAG;
- AI-agent orchestration;
- local/cloud model abstraction;
- human-in-the-loop;
- RBAC;
- auditability;
- Docker/Linux deployment.

## 5. Цільові користувачі MVP

### Production Manager
Переглядає ризики виробничого плану та підтверджує запропоновані дії.

### Procurement Specialist
Отримує чернетку задачі щодо дефіцитного компонента.

### Engineer
Переглядає технічні документи та допустимі альтернативи.

### AI Administrator
Контролює моделі, запуски, помилки та політики.

### Auditor
Переглядає execution trace без права змінювати дані.

## 6. Межі проєкту

### У MVP входить

- synthetic ERP dataset;
- один Golden Scenario;
- risk calculation engine;
- engineering RAG;
- один контрольований agent workflow;
- human approval;
- audit log;
- dashboard;
- authentication і базовий RBAC;
- Docker Compose;
- публічний deployment на VPS;
- автоматичні тести;
- portfolio documentation.

### У MVP не входить

- керування фізичними роботами;
- бойові сценарії;
- реальна телеметрія;
- інтеграція з реальним ERP;
- довільний no-code конструктор агентів;
- повноцінний BPMN editor;
- мультиагентний “рой” без детермінованого control flow;
- мобільний застосунок;
- production-grade high availability;
- billing;
- підтримка багатьох організацій;
- fine-tuning моделей;
- автоматичне затвердження закупівель.

## 7. Критерій успіху

Проєкт успішний, якщо сторонній технічний спеціаліст за 10–15 хвилин може:

1. зрозуміти бізнес-проблему;
2. запустити або відкрити систему;
3. пройти Golden Scenario;
4. побачити обґрунтовані ризики та джерела;
5. підтвердити одну дію;
6. відкрити audit trace;
7. переконатися, що рішення не є набором статичних моків.

## 8. Власник продукту

Власник продукту та фінальний acceptance authority — автор проєкту.

AI-агенти можуть пропонувати рішення, писати код, створювати тести й документацію, але не мають права самостійно змінювати Definition of Done.
